---++ Description

This plugin is used to process XML data using XPath or XSLT. There are two distinct use cases for this plugin:

   * picking a few values from an xml database
   * transform xml data into a suitable way

For the first use case xpath is the easiest way to come to good results quickly. This approach starts
at a given set of nodes and formats their properties the Foswiki way (header, format, separator, footer).
Note however the limitations given that xml data may be more complex that this approach allows to access it.
In this case xslt is more flexible reformating data as required. 

Note that the xslt processor does not allow to import or export any data from within an xslt itself for security
reasons. It may only be used to transform the given xml data and only use the xslt file provided.

---++ Syntax

=%<nop>XML{"&lt;xml-attachment>" [params]}%=

| *Parameter* | *Description* | *Default* |
| "..." or attachment="..." | name of the xml file attached to a topic | | 
| topic="..." | topic where the xml file is attached to | current topic | 
| raw="on/off" | output the xml file as is | off | 
| xpath="..." | xpath expression to find nodes | | 
| xslt="..." | xslt string provided inline to process the xml file with | | 
| xslt_attachment="..." | name of the xslt file attached to a topic being used to process the xml data | | 
| xslt_topic="..." | topic where the xslt file is attached to | see topic parameter | 
| &lt;customParameter>="..." | custom parameter(s) that are passed to the xslt transformer | | 
| header="...", format="...", separator="..." footer="..." | format strings used in xpath mode | |
| skip="..." | in xpath mode number of nodes to skip | _none_ |
| limit="..." | in xpath mode number of nodes to format | _all_ |

Note that the macro operates in either raw, xslt or xpath mode. In xpath mode the format strings
header, format, separator, footer are used. In xslt mode the output of the transformation is being inserted.
For each node found using the =xpath= parameter, the =format= string is used to output results with following variables:

   * =$index=: running index of found nodes
   * =$nodeName=: name of the current node 
   * =$nodeValue=: value of the node
   * =$nodeType=: node type
   * =$textContent=: text within the current node
   * =$find(...)=: a subsequent xpath expressions to find more values starting at the current node
   * =$encode(...)=: entity encode a string

---++ Examples

<verbatim>
%XML{"books.xml" xslt_attachment="table_of_books.xslt"}%
</verbatim>

<verbatim>
%XML{"books.xml" xslt_attachment="display_book.xslt" book_id="%URLPARAM{"id"}%"}%
</verbatim>

---++ XSLT extensions

Some Foswiki APIs have been registered to XSLT into the =foswiki= namespace.

| *Function* | *Description* |
| =entityEncode(_arg1, arg2, ..._)= | entity encodes all args and concatentates them |
| =entityDecode(_arg1, arg2, ..._)= | |
| =urlEncode(_arg1, arg2, ...)= | |
| =urlDecode(_arg1, arg2, ...)= | |
| =isTrue(_arg_)= | returns 1 if the argument is true-ish | 
| =getScriptUrl(_web_, _topic_, _script, _arg1_, _arg2_, ..._)= | |
| =getScriptUrlPath(_web_, _topic_, _script, _arg1_, _arg2_, ..._)= | |
| =getPubUrlPath(...)_= | |
| =getPreferencesValue(_key_, _[web]_)= | |
| =getPreferencesFlag(_key_, _[web]_)= | |
| =extract(_text_, _pattern_)= | extracts a _pattern_ in the _text_; the _pattern_ must include at least one bracket group |
| =subst(_text_, _pattern_, _with_, _flags_)= | replaces _pattern_ in the _text_ with a substitute; optional _flags_ can be "g", "gi" or "i" as used in perl's regex processor |

To make use of them you must register the namespace in the xml preamble. Here is an example of how to use =urlEncode=:

<verbatim>
<xsl:stylesheet version='1.0' 
   xmlns:xsl='http://www.w3.org/1999/XSL/Transform' 
   xmlns:foswiki="urn:foswiki">
...
   <a>
      <xsl:attribute name="href">
         <xsl:value-of select="concat('?id=', foswiki:urlEncode(PRODUCT_ID))" />
      </xsl:attribute>
      ...
   </a>
...
</xsl:stylesheet>
</verbatim>
